PURPOSE:
  Solve the 166-trade backtester problem where most trades had empty fields
  (entry_price=0, question='', direction=''). Provides complete audit trail.

FILES DEPLOYED:

  Export:     /opt/polybot/state/trade_log.jsonl (JSON lines)
  Logs:       /opt/polybot/logs/trade_logger.log

ARCHITECTURE:
  - SQLite database with 2 tables: trades, exits
  - Dual logging: SQLite + JSONL (redundancy + portability)
  - Full context capture: market_id, condition_id, question, direction,
    entry_price, position_size_usd, shares_bought, kelly_fraction_used,
    intelligence_score, strategy_category, reasoning
  - Exit tracking: exit_price, exit_timestamp, exit_reason, holding_period,
    realized_pnl_usd, realized_pnl_pct

PYTHON API:
  from trade_logger import TradeLogger
  tl = TradeLogger()

  # Log trade
  trade_id = tl.log_trade(market_id, condition_id, question, direction,
                          entry_price, position_size_usd, shares_bought,
                          kelly_fraction_used, intelligence_score,
                          strategy_category, reasoning)

  # Log exit
  exit_id = tl.log_exit(trade_id, exit_price, exit_reason,
                        holding_period_hours, realized_pnl_usd, realized_pnl_pct)

PIPELINE ARCHITECTURE (15-minute cycle):

  6. MONITOR   → Check all open positions and resolutions
  7. EXIT      → Check position_lifecycle for exit conditions
  8. LOG       → Update trade_logger and attribution
  9. ALERT     → Send Discord alerts
