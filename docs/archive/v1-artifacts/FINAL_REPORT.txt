1. CORE ENGINE (strategy_attribution.py — 488 lines)
   
   Classifies every position and trade into 8 strategy categories:
   • weather-temp (temperature predictions)
   • weather-precip (rain/snow/wind)
   • crypto-price (BTC/ETH movements)
   • crypto-event (forks/upgrades)
   • politics (elections/votes)
   • sports (game outcomes)
   • finance (inflation/rates/earnings)
   • other (unclassified)

   Computes 18+ metrics per strategy:
   • Capital deployed, positions, unrealized/realized P&L
   • Win rate, average win/loss, profit factor
   • Sharpe ratio, max single loss, volatility
   • Duration, trend analysis

   Outputs:
   • Human-readable text report (printed to stdout)
   • JSON file for machine consumption
   • Zero external dependencies (stdlib only)

2. HTTP SERVICE (attribution_service.py — 111 lines)
   
   Serves attribution data via REST API:
   • GET /api/attribution → returns current attribution JSON
   • GET /health → status check
   • Runs on port 5001
   • CORS enabled
   • Auto-reload data on each request
   • Systemd service with auto-restart

3. CRON AUTOMATION
   
   Runs every hour at :30 minutes:
   • Executes strategy_attribution.py
   • Updates /opt/polybot/state/strategy_attribution.json
   • Logs to /tmp/attribution_cron.log
   • Zero manual intervention required
