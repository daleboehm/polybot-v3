# VPS Deployment Changes — 2026-03-25

## Overview
Updated Polymarket sprint_trader on DigitalOcean VPS (178.62.225.235) to add stale order cancellation logic and increase risk/position limits.

## Changes Made

### 1. `/opt/polybot/sprint_trader.py`

#### Added `cancel_stale_orders(client)` function (before `execute_trades`)
- **Purpose**: Cancel orders with zero fills (size_matched == 0) to free committed capacity
- **Behavior**:
  - Reads `/opt/polybot/state/open_orders.json`
  - Iterates through open orders, finds those with `size_matched == 0`
  - Calls `client.cancel(order_id)` for each stale order
  - Prints cancellation status and freed dollar amount
  - Returns total freed dollars
- **Error Handling**:
  - Gracefully handles missing `open_orders.json`
  - Catches and logs cancel failures individually
  - Does not crash if a single order fails

#### Modified `execute_trades()` initialization
- Added call to `cancel_stale_orders(client)` immediately after CLOB client initialization
- Placement: After client setup, before "LIVE EXECUTION MODE" header
- Runs BEFORE any new orders are placed
- Freed capacity is printed for visibility

### 2. `/opt/polybot/run_sprint.sh`

## Implementation Details

### cancel_stale_orders Logic
```
1. Check for /opt/polybot/state/open_orders.json existence
2. Load JSON (with error handling)
3. For each order in the list:
   - Extract: order_id, size_matched, dollar_value, question
   - If size_matched == 0 and order_id exists:
     - Call client.cancel(order_id)
     - Accumulate freed_dollars
     - Log success/failure
4. Print summary: count of cancelled orders + total freed
5. Return freed_dollars for display in execute_trades
```

### Impact on Trading Flow
1. VPS cron triggers `/opt/polybot/run_sprint.sh` every 15 minutes
2. Script runs sprint_trader with new --risk 75 / --per-trade 20
3. `execute_trades()` is called with max_risk=75, max_per_trade=20
4. BEFORE placing new orders, cancel_stale_orders executes:
   - Frees up any zero-fill orders from previous runs
   - Reclaims committed but unfilled capacity
5. Freed capacity is reported to logs
6. New trades placed with updated limits
