DEPLOYED SCRIPTS (on VPS at /opt/)
================================

/opt/deploy_entity.sh
  Create a new isolated trading entity
  Usage: ./deploy_entity.sh <entity_name> <dashboard_port> <cron_offset>
  
  Parameters:
    entity_name     - Unique identifier (e.g., "lilac")
    dashboard_port  - HTTP port (1024-65535, must be unique)
    cron_offset     - Minutes offset for scheduling (0-59)

/opt/list_entities.sh
  List all active trading entities
  Usage: ./list_entities.sh
  Output: Entity names, wallets, ports, status, equity

=====================

# Deploy entity "lilac" on port 8081 with cron offset 2 minutes
/opt/deploy_entity.sh lilac 8081 2

# Deploy entity "arbitrage-bot" on port 8090 with cron offset 10 minutes

WHAT GETS CREATED
=================

For each entity, the script:
  1. Creates /opt/<entity_name>/ directory
  2. Copies all 11 Python trading scripts
  3. Copies dashboard web UI
  4. Symlinks shared Python venv (saves disk)
  5. Auto-generates unique Polygon wallet
  6. Initializes portfolio state (zero balance)
  7. Adds 11 cron jobs (offset-scheduled)
  8. Starts web dashboard on dedicated port

Result: Fully operational trading entity in ~30 seconds

  3. Monitor first run: tail -f /opt/<name>/logs/reconcile.log
  4. Access dashboard: http://178.62.225.235:<port>

CRON SCHEDULE (per entity)
==========================

All jobs offset by <cron_offset> minutes to avoid resource contention

Every 5 min:    reconcile, sprint_trader, quick_cycle, arb_executor,
                position_monitor, correctness_verifier, open_orders_cache,
                auto_trader

Every 30 min:   auto_redeem
Hourly:         health_monitor
At :05 & :35:   auto_deposit

Example with offset=2:
  :02 - first 5-min cycle
  :07 - second 5-min cycle
  :12 - third 5-min cycle
  etc.

TROUBLESHOOTING
===============

Dashboard not accessible?
  1. Check if process is running: ps aux | grep dashboard
  2. Verify port: netstat -tulpn | grep :<port>
  3. View logs: tail -f /opt/<entity_name>/logs/dashboard_<port>.log
  4. Restart: kill <pid> && cd /opt/<entity_name> && python3 dashboard.py --port <port> &

Wallet not syncing?
  1. Check reconcile.py logs: tail -f /opt/<entity_name>/logs/reconcile.log
  2. Verify wallet has USDC: Check blockchain directly
  3. Force reconciliation: cd /opt/<entity_name> && python3 reconcile.py

Entity won't deploy?
  1. Check if entity_name already exists: ls -d /opt/<name>
  2. Verify port is available: netstat -tulpn | grep :<port>
  3. Check venv: ls -la /opt/polybot/venv/bin/python3
  4. Run with verbose: bash -x /opt/deploy_entity.sh ...


PARAMETERS QUICK GUIDE
======================

Entity Name Rules:
  - Unique (no entity with same name)
  - Lowercase letters, numbers, hyphens only
  - Example: lilac, caspian-intl, bot-001
  - Becomes directory name at /opt/<entity_name>/

Dashboard Port Rules:
  - Unique (no two entities on same port)
  - Valid range: 1024-65535
  - Recommended: 8080+ (above armorstack's 8080)
  - Use netstat to check: netstat -tulpn | grep LISTEN

Cron Offset Rules:
  - 0-59 minutes
  - Spreads jobs to avoid thundering herd
  - Example: 0, 2, 4, 6, 10 (staggered)
  - For N entities, use: 2*i for i in 0,1,2...
